## 模拟面试系统综合实训总结报告

### 一、实训背景与目的

随着企业对技术人才要求不断提高，求职面试成为每位计算机专业学生必须面临的挑战。为了帮助求职者更好地准备面试、提升面试表现，本次实训选择基于 **RuoYi-Vue（若依）** 企业级开发框架，开发一套**模拟面试管理系统**。

RuoYi-Vue 是当前国内最流行的 Spring Boot + Vue 前后端分离后台管理框架之一，广泛应用于各类企业级项目。本次实训的核心任务是：以该框架为基础，扩展开发一套完整的模拟面试功能模块，实现面试岗位管理、AI自动出题、用户面试作答、AI智能评分、面试报告生成等全流程功能。

📷 [在此插入：系统整体架构图]

### 二、项目概述

本项目是一个基于 **RuoYi-Vue v3.9.2** 扩展的模拟面试管理平台，系统主要服务于微信小程序端用户，支持用户选择岗位进行模拟面试、AI自动出题、回答评分、生成面试报告等功能。

**系统架构**：
- **后端**：Spring Boot 2.x + MyBatis + MySQL + Redis + Spring AI
- **前端**：Vue 2 + Element UI + Vuex + ECharts
- **AI能力**：Spring AI + DeepSeek 大模型
- **权限模型**：RBAC（基于角色的访问控制）

📷 [在此插入：项目技术栈架构图]

#### 2.1 项目主要功能模块

- **面试岗位模块**：管理不同岗位类型（Java开发、前端开发、数据分析等），支持排序与状态控制
- **面试维度模块**：每个岗位可配置多个评分维度，如基础知识、项目经验、沟通能力、技术深度等，支持权重自定义
- **面试记录模块**：记录用户面试全过程（开始时间、结束时间、持续时间、状态）
- **面试问题模块**：支持AI自动出题，按岗位和维度生成针对性的面试题目
- **面试回答模块**：支持文本回答与语音回答上传
- **维度评分模块**：基于AI对各维度进行评分
- **面试报告模块**：生成综合评分报告，包含优势分析、不足分析与改进建议
- **小程序用户模块**：微信用户管理与登录认证

### 四、遇到的问题与解决方案

#### 4.1 RuoYi-Vue框架理解门槛
初次接触若依框架，其Maven多模块结构和大量配置让人一头雾水。解决方案：从官方文档入手，逐个模块分析职责，边调试边理解，最终掌握了其核心架构思想。

#### 4.2 代码生成器的二次开发
若依代码生成器生成的代码是标准的CRUD模板，但面试模块的业务逻辑较为复杂（状态流转、权重计算、多表关联）。解决方案：在生成的代码基础上进行深度定制，手动编写业务逻辑层。

#### 4.3 AI模型集成
Spring AI + DeepSeek的集成过程中，遇到了依赖版本冲突、API调用参数配置等问题。解决方案：查阅官方文档，调整pom.xml中依赖版本，正确配置API密钥和模型参数。

#### 4.4 前端路由与权限配置
自定义模块需要配置菜单路由和权限标识。解决方案：按照RuoYi-Vue的权限体系规范，在前端配置路由元信息（meta），在后端配置权限注解（@PreAuthorize）。
